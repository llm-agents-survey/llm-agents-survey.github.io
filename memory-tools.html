Memory, Tools & Actions

How agents remember, what they can do, and where they can act

Overview

If reasoning and planning are the mind of an agent, memory, tools, and action spaces are its senses and hands. These three pillars define what an agent can know and do:

  • Memory: Extends what an agent can remember beyond its context window
  • Tools: Extends what an agent can do beyond generating text
  • Actions: Defines the environments where the agent can operate

The evolution here has been striking: from simple web search tools (2022) to computer-use agents controlling full desktop environments (2024-2025).

Memory Systems

The Memory Problem

LLMs have finite context windows. An agent handling a long task, managing a long-running conversation, or needing to learn from experience requires memory architectures that go beyond the prompt.

Memory Types Taxonomy

Based on cognitive science (ACT-R, SOAR) and the agent literature:

Type Description Implementation
Working memory Current context, immediate task In-context window
Episodic memory Records of past events External store with timestamps
Semantic memory World knowledge, facts Vector DB, knowledge graph
Procedural memory Skills, how-to knowledge Code libraries, tool descriptions

MemGPT: Towards LLMs as Operating Systems (2023)

Packer et al. · arXiv:2310.08560

Inspired by OS virtual memory management. MemGPT manages a hierarchy of memory tiers — fast (in-context) and slow (external storage) — and moves information between them via interrupt-like function calls. Enables indefinitely long conversations and document analysis.

  • Key ideas: Virtual context management; self-directed memory management; “paging” information in and out of context
  • Result: Outperforms baselines on long-document analysis and multi-session conversations
  • GitHub: cpacker/MemGPT → evolved into Letta

Generative Agents Memory Architecture (2023)

Park et al. · arXiv:2304.03442

Three-layer memory for the Smallville agent simulation:

  1. Observation store: Raw events recorded as natural language
  2. Reflection: Periodic synthesis of observations into higher-level insights
  3. Planning: Memory-informed scheduling and goal setting

Retrieval weighted by recency, importance, and relevance. Sets the standard for agent memory design.

ReadAgent: Gist Memory for Long Documents (2024)

Lee et al. (Google) · arXiv:2402.09727

Human-inspired reading: divide long documents into episodes, compress into “gist memories,” dynamically retrieve detailed passages when needed. Extends effective context 3.5–20× without retraining.

Cognitive Architectures for Language Agents (2023)

Sumers et al. · arXiv:2309.02427

Grounds agent architecture in cognitive science. The most rigorous taxonomy of memory types in the context of LLM agents. Required reading for memory system design.

A-MEM: Agentic Memory System (2025)

Dynamic memory system that structures memories like a Zettelkasten — interconnected notes with evolving knowledge structures rather than flat retrieval.

Mem0 (Production Memory)

github.com/mem0ai/mem0 — 25k+ stars

Production-ready memory layer for AI agents. Manages user-specific memory across sessions. Supports OpenAI, Anthropic, and open models. Widely used in production deployments.

ENGRAM (2025)

arXiv:2511.12960

Lightweight memory orchestration supporting episodic, semantic, and procedural memory with normalized schemas. Designed for long-horizon conversational agents.

Tool Use & Function Calling

The Tool Use Revolution

The shift from text generation to tool use transformed LLMs from knowledge retrievers into action takers. Key milestones:

  • Toolformer (Feb 2023): First principled approach to self-supervised tool learning
  • OpenAI function calling (Jun 2023): Made tool use practical and reliable at scale
  • Claude tool use (2023): Anthropic’s structured tool calling API

Toolformer (2023)

Schick et al. · arXiv:2302.04761

Self-supervised approach: the model learns to insert API calls into its own generated text and incorporate results. Minimal human annotation. Learned tools: calculator, Wikipedia, calendar, QA, translator.

  • Significance: Proved that tool learning can be learned, not just prompted

OpenAI Function Calling (June 2023)

Not a paper — a product release — but arguably the most impactful event of 2023 for practitioners. Structured tool calling via JSON schema definitions, with guaranteed structured outputs.

  • Impact: Enabled reliable, production-grade tool use; became the de facto industry standard; adopted by Anthropic, Mistral, Llama, and others

Gorilla: Large Language Model Connected with Massive APIs (2023)

Patil et al. (UC Berkeley) · arXiv:2305.15334

Fine-tuned model trained to write correct API calls from 1,600+ APIs across TorchHub, TensorFlow Hub, HuggingFace. Outperforms GPT-4 on API selection accuracy.

  • GitHub: ShishirPatil/gorilla

ToolLLM: Facilitating LLMs to Master 16000+ Real-World APIs (2023)

Qin et al. · arXiv:2307.16789

ToolBench: 16,000+ real-world REST APIs. Training data via ChatGPT. DFSDT (depth-first search decision tree) for multi-step tool use. ToolEval benchmark.

HuggingGPT / JARVIS (2023)

Shen et al. · arXiv:2303.17580

LLM (ChatGPT) as orchestrator of hundreds of HuggingFace models. Task planning → model selection → execution → result synthesis. Early vision of “models as tools.”

RestGPT (2023)

Song et al. · arXiv:2306.06624

Connects LLMs to real-world RESTful APIs. Coarse-to-fine online planning with an API executor that handles paginated responses and complex schemas.

Large Language Models as Tool Makers (2023)

Cai et al. · arXiv:2305.17126

LLMs don’t just use tools — they create them. A “tool maker” generates reusable Python functions; a “tool user” invokes them. Tool library grows over time.

CodeAct: Executable Code as Agent Actions (2024)

Wang et al. · arXiv:2402.01030

Instead of structured function calls, the agent generates executable Python code as its action. More expressive than JSON tool calls; enables complex multi-step operations in a single action.

  • Key insight: Code is a universal action language; execution feedback enables correction

AnyTool (2024)

Du et al. · arXiv:2402.04253

Self-reflective API selector that recursively decomposes tool retrieval. Works with thousands of APIs using hierarchical retrieval.

Action Spaces

Web Browsing Agents

WebGPT (2022) · arXiv:2112.09332
First web-browsing agent. GPT-3 fine-tuned with RLHF to search the web and answer long-form questions with citations.

Mind2Web (2023) · arXiv:2306.06070
Towards generalist web agents. 2,000 tasks across 137 real websites. Covers 3 action types: click, type, select. Evaluation for cross-website and cross-task generalization.

WebArena (2023) · arXiv:2307.13854
Realistic web environments (4 functional sites: e-commerce, Reddit, GitLab, map). 812 tasks. Tests true multi-step web navigation.

WebAgent (Google, 2023) · arXiv:2307.12856
Two LLMs: one for planning, one for HTML grounding. Generates Python programs to complete web tasks. 65% on MiniWoB++.

GUI & Computer Use Agents

CogAgent (2024) · arXiv:2312.08914
18B visual language model for GUI agents. High-resolution image understanding for web pages and desktop GUIs. Achieves SOTA on Mind2Web and AITW.

AppAgent (2024) · arXiv:2312.13771
Multimodal agent for smartphone app control. Autonomous exploration to learn app interfaces; builds a knowledge base of UI interactions.

UFO: A UI-Focused Agent for Windows OS Interaction (2024)
arXiv:2402.07939 · Microsoft
Dual-agent framework for Windows: one agent selects the application, another controls it. Uses GPT-Vision for screenshot understanding.

OS-Copilot (2024) · arXiv:2402.07456
Self-improving desktop agent. Learns new OS skills through experience; builds reusable skill library. Evaluated on GAIA benchmark.

Anthropic Computer Use (October 2024)
Claude natively controls desktop computers: moves mouse, types, takes screenshots, navigates GUIs. First major commercial deployment of computer-use capability.

OSWorld Benchmark (2024) · arXiv:2404.07972
369 real-world desktop tasks across Ubuntu/Windows/macOS. Models scored <10% at release (2024); specialized models pushed above 70% by 2025.

Code Execution Agents

OpenDevin / OpenHands (2024)
github.com/All-Hands-AI/OpenHands — 40k+ stars
Open-source autonomous software engineer. Full sandboxed execution environment: file editing, web browsing, code running. Supports multiple LLM backends.

SWE-agent (2024) · arXiv:2405.15793
Agent-Computer Interface (ACI) for GitHub issue resolution. Custom interface optimized for repo navigation and code editing. 12.29% on SWE-bench Verified at release.

Robotics & Embodied Agents

SayCan (2022) · arXiv:2204.01691
Combines LLM planning with robot affordance models. Language proposes; physical feasibility scores filter. “What can I do that is both useful AND physically possible?”

RT-2 (2023) · arXiv:2307.15818
Vision-Language-Action (VLA) model from Google. End-to-end: from image + instruction to robot motor commands. Emergent reasoning about novel objects.

PaLM-E (2023) · arXiv:2303.03378
562B multimodal language model for embodied reasoning. Integrates visual observations directly into language model reasoning for robot planning.

Retrieval-Augmented Agents

Self-RAG (2023)

Asai et al. · arXiv:2310.11511

Self-reflective retrieval: the model decides whether to retrieve (not always), evaluates retrieved passages for relevance and support, and critiques its own outputs. Outperforms standard RAG and always-retrieve baselines.

Agentic RAG Patterns (2024)

Key patterns from practitioner literature:

  • Query planning: Decompose complex questions into sub-queries
  • Multi-hop retrieval: Retrieve → reason → retrieve again
  • Self-correction: Evaluate retrieved context quality; re-retrieve if insufficient
  • Knowledge graph integration: Structured traversal alongside vector retrieval
